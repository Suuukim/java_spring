spring MVC 프로젝트

1. MVC 디자인패턴
: 웹애플리케이션 개발시 사용됩니다.
Model	: 컨트롤러와 view 사이에서의 데이터 전달을 위한 객체
View	: 사용자 화면(인터페이스) 파일. 우리는 jsp파일
Controll	: 사용자 요청을 처리하는 클래스

2. spring의 컨트롤러
: DispatcherServlet 은 가장 앞단에서 모든 요청을 처리하는
frontcontroller 입니다. 프론트 컨트롤러가 구체적인 컨트롤러로 요청에
대한 처리를 위임합니다.
: @Controller 어노테이션으로 표시하여 bean으로 생성됩니다.
컨트롤러 클래스의 메소드는 각각 url과 매핑되어 요청을 처리합니다.
메소드의 인자는 요청의 파라미터를 받을 수 있도록 선언할 수 있습니다.
메소드의 매개변수(인자)가 클래스 타입일때는 요청파라미터와 자동으로 
매핑됩니다.
: 요청과 매핑되는 메소드의 return 은 view 파일 지정 또는
redirect 되는 url 입니다. view 파일명이 url과 같을 때는 생략합니다.
